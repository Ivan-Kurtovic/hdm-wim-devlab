SemRep Projekt 1

[Endpoints mit Jersey]

Dieses Projekt wird auf der AppengineFlexible ausgeführt und bietet verschiedene Endpoints an. Das Projekt ist ein Studienprojekt und stellt einen Prototyp dar. Der Prototyp stellt anderen Komponenten bzw. Gruppen über verschiedene Endpoints Daten zur Verfügung. Diese Daten sind in einer Ontologie hinterlegt und werden mit dem Jena Framework manipuliert. Dieses Projekt funktoniert ohne PubSub. Das gleiche Projekt mit Google Pub Sub Integration finden Sie neben weiteren Informationen auf dem GitHub-Repository des Studienprojekts:
  • GitRepository

GUI

Diese Komponente ist für die Anzeige der abgeleiteten Dokumentenvorschläge verantwortlich. Zudem stellt die Gruppte eine Dokumenthistorie zur Verfügung, die zuvor aufgerufte Dokumente eines Nutzer enthält.In diesem Testfall sind statische Keywörter als Demodaten hinterlegt.
Endpoints:
  • getDocumentOffers
  • getAllDocuments

Event Interface

Dem EventInferface stellen wir Getter zur verfügung, so dass diese Daten aus der Ontologie abfragen können.Auf Basis der Daten werden von der Komponente komplexe Events gebaut.In diesem Testfall sind statische Keywörter als Demodaten hinterlegt.
Endpoints:
  • getUserInformation
  • getDocumentInformation
  • ProjectInformation
  • getDepartmentInformation
  • getAllProjects
  • getAllProjectRoles
  • getAllDepartments
  • getAllCompanies

Tokenizer Interface

Das Tokenizer Interface wird benutz um Metadaten die sich in der Ontologie auf Basis von Keywörter zuzuordnen. In diesem Testfall sind statische Keywörter als Demodaten hinterlegt.
  • getTokenizerMetadata

Internalinterface

Das Internal Interface wird von dem Semrep-GoogleAddin benutzt. Mit diesem lassen sich Metadaten generieren. Das Addin liest alle Projekte, Dokumentenkategorien und Projektpahsen aus. Sind Metadaten zum Dokument über ads Addin eingefügt, so können diese in der Ontologie aktualiert und gespeichert werden.
Endpoints:
  • getdoccategory
  • getprojects
  • getprojectstage
  • insertMetadata
